Информационно-аналитическая система управления заказами кофейни
О проекте
Проект посвящен разработке информационно-аналитической системы для автоматизации обработки заказов в кофейне и отслеживания их статусов на всех этапах выполнения.
Система предназначена для централизованного управления заказами, поступающими из различных каналов: кассы, мобильного приложения, сайта и терминалов самообслуживания.
В рамках проекта были проанализированы существующие бизнес-процессы, сформированы требования к системе, спроектирована структура данных и разработана модель взаимодействия между участниками процесса.
________________________________________
Цель проекта
Разработать концепцию системы, которая позволит:
•	автоматизировать обработку заказов;
•	отслеживать статус заказа в режиме реального времени;
•	сократить количество ошибок при выполнении заказов;
•	обеспечить контроль работы персонала;
•	получать аналитическую отчетность по работе кофейни.
________________________________________
Бизнес-проблема
На момент анализа обработка заказов выполняется через несколько каналов продаж и требует постоянного взаимодействия сотрудников.
Основные проблемы:
•	отсутствие единого источника информации о заказах;
•	сложность отслеживания текущего статуса заказа;
•	ошибки при передаче заказов между сотрудниками;
•	отсутствие данных о времени приготовления и загрузке персонала;
•	ограниченные возможности для анализа эффективности работы.

Выполненные задачи
Анализ требований
•	анализ предметной области;
•	выявление проблем существующего процесса;
•	сбор и формализация бизнес-требований;
•	подготовка функциональных и нефункциональных требований.
Моделирование процессов
•	построение BPMN-диаграмм AS-IS;
•	построение BPMN-диаграмм TO-BE;
•	описание целевого процесса обработки заказов.
Проектирование системы
•	разработка пользовательских сценариев (User Story);
•	разработка вариантов использования (Use Case);
•	проектирование логической модели данных;
•	подготовка ER-диаграммы.
Проектирование интеграций
•	описание REST API;
•	подготовка контрактов взаимодействия между сервисами;
•	описание структуры запросов и ответов.
Аналитическая отчетность
•	определение ключевых показателей эффективности;
•	разработка SQL-запросов для формирования отчетов;
•	описание требований к аналитическим отчетам.
